# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Build (runs templ generate + Tailwind CSS + go build)
make build

# Run dev server with live reload (uses air)
PORT=8080 make watch

# Run without live reload
PORT=8080 make run

# Run all tests
make test

# Run a single test
go test ./internal/server/... -run TestHealthHandler -v
```

## Environment

Requires a `.env` file (already present) with:
- `PORT` — HTTP listen port
- `BLUEPRINT_DB_URL` — SQLite file path (e.g. `./test.db`)
- `APP_ENV` — environment name

The dev environment is managed by **flox** (`.flox/`). Run `flox activate` to
enter it (or prefix commands: `flox activate -- make build`). It provides go,
templ, tailwindcss, and air. The SQLite driver is pure Go (`modernc.org/sqlite`),
so builds need no CGO/C toolchain.

## Editing site content

All page content (bio, tagline, location, social icons, featured video, link
buttons) lives in **`content/content.yaml`** — not in Go code. To change the
page or add a link, edit that file and rebuild; committing to `main` redeploys
automatically. The `content` package embeds the YAML at build time and parses
it once into `content.Site`. `content/content_test.go` validates the file in
CI, so a malformed edit fails the build instead of deploying a broken page.

## Architecture

**Entry point:** `cmd/api/main.go` — starts the Echo HTTP server with graceful shutdown.

**Content** (`content/`):
- `content.yaml` — the editable source of truth for everything on the page.
- `content.go` — embeds `content.yaml` (`//go:embed`), defines the structs, and exposes `content.Site` (parsed once at startup; panics on malformed YAML).

**Server layer** (`internal/server/`):
- `server.go` — constructs `Server` (port + DB), wires up `http.Server`
- `routes.go` — registers Echo routes and serves the `QuickLinks` templ component with `content.Site`. Both `/` and `/links` serve the same page. Unknown routes redirect to `/`. Also sets HSTS when behind an HTTPS proxy.

**Templates** (`cmd/web/`):
- `.templ` files are the source; `_templ.go` files are auto-generated by `templ generate` — never edit them directly.
- `efs.go` embeds `assets/` (CSS, JS) into the binary via `embed.FS`.
- Tailwind input is `cmd/web/styles/input.css`; output goes to `cmd/web/assets/css/output.css`.

**Database** (`internal/database/`):
- SQLite via `go-sqlite3`. Currently only used for the `/health` endpoint. The `Service` interface makes it easy to mock in tests.

**Build order matters:** `templ generate` → Tailwind CSS → `go build`. The `make build` target handles this. Air (live reload) also calls `make build` on file changes.
